# دليل ترقية EduPay للإنتاج — معالجة نقاط الضعف الأمنية والمعمارية

هذا الدليل يحوّل النموذج الأولي (HTML في المتصفح) إلى منتج إنتاجي آمن على **Supabase + Next.js**.
كل نقطة ضعف في التقييم لها حلّ محدد هنا.

---

## كيف تُعالَج نقاط الضعف الأربع

| نقطة الضعف (النموذج الحالي) | الحل الإنتاجي | الملف |
|---|---|---|
| **المصادقة وهمية** (تُتجاوز من المتصفح) | Supabase Auth — مصادقة حقيقية بـ JWT على الخادم | إعداد Supabase |
| **كلمات المرور مكشوفة** (نص صريح) | Supabase Auth يخزّنها مشفّرة (bcrypt)، لا تلمسها أبداً | إعداد Supabase |
| **العزل بين المدارس تجميلي** | RLS — العزل يُفرض في قاعدة البيانات على كل استعلام | `02_rls_policies.sql` |
| **لا بيانات دائمة** (ذاكرة فقط) | جداول PostgreSQL دائمة | `01_schema.sql` |
| **المدفوعات محاكاة** | ربط Thawani (عُمان) أو Stripe | قسم الدفع أدناه |
| **استعادة كلمة المرور صورية** | Supabase Auth — إعادة تعيين عبر البريد جاهزة | إعداد Supabase |

## الخطوات بالترتيب

### 1) إنشاء مشروع Supabase
- أنشئ حساباً على supabase.com (الباقة المجانية تكفي للبداية)
- أنشئ مشروعاً جديداً، اختر أقرب منطقة (مثلاً Frankfurt للخليج)
- احفظ `Project URL` و `anon key` و `service_role key`

### 2) تنفيذ المخطط وسياسات الأمان
في لوحة Supabase → SQL Editor، نفّذ بالترتيب:
1. `01_schema.sql` — الجداول والعلاقات
2. `02_rls_policies.sql` — سياسات العزل والصلاحيات (**الأهم أمنياً**)
3. `03_functions.sql` — دوال التسجيل والتأمينات والموافقات

### 3) تفعيل المصادقة
- Authentication → Providers → فعّل Email
- فعّل "Confirm email" لاستعادة كلمة المرور
- استعادة كلمة المرور تعمل تلقائياً عبر `supabase.auth.resetPasswordForEmail()`

### 4) التخزين (الشعارات والإيصالات)
- Storage → أنشئ bucket باسم `logos` و `receipts`
- ارفع الشعارات والإيصالات كملفات (لا DataURL) — أخف وأأمن

### 5) واجهة Next.js
هيكل مقترح:
```
app/
  (auth)/login/page.tsx          ← تسجيل الدخول
  (auth)/register/page.tsx       ← تسجيل المدرسة
  dashboard/page.tsx             ← لوحة المدير
  students/page.tsx              ← سجل الطلاب
  employees/page.tsx             ← الموظفون + الرواتب
  fees/page.tsx                  ← الرسوم والفواتير
  subscription/page.tsx          ← اشتراك المنصة
lib/
  supabase.ts                    ← عميل Supabase
components/                      ← المكوّنات المشتركة
```

### 6) ربط الدفع الحقيقي
- **عُمان**: بوابة Thawani (الأكثر استخداماً محلياً) — لديها API وSDK
- **الخليج**: Stripe أو Tap Payments
- التحويل البنكي: يبقى كما هو (رفع إيصال → اعتماد المدير)، لكن الإيصال يُرفع لـ Storage

## مثال: عميل Supabase آمن (lib/supabase.ts)

```typescript
import { createClient } from '@supabase/supabase-js'

export const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
)
// ملاحظة: anon key آمن للعرض — RLS يحمي البيانات، لا المفتاح
```

## مثال: تسجيل الدخول الحقيقي

```typescript
const { data, error } = await supabase.auth.signInWithPassword({
  email, password
})
// لا تحقق في المتصفح — Supabase يتحقق على الخادم ويصدر JWT
```

## مثال: جلب طلاب المدرسة (RLS يطبّق العزل تلقائياً)

```typescript
const { data: students } = await supabase
  .from('students')
  .select('*')
// لا حاجة لـ where school_id — RLS يضيفه تلقائياً ويمنع رؤية مدارس أخرى
```

## مثال: تسجيل مدرسة جديدة

```typescript
// 1) إنشاء المستخدم
await supabase.auth.signUp({ email, password })
// 2) إنشاء المدرسة + المدير + التجربة (دالة الخادم)
await supabase.rpc('register_school', {
  p_name, p_branch, p_country, p_currency,
  p_cr, p_license, p_vat, p_phone, p_email, p_address, p_owner_name
})
```

## مثال: اعتماد طلب راتب (المدير فقط — يفرضه الخادم)

```typescript
await supabase.rpc('approve_salary_request', { p_request_id })
// لو حاول محاسب استدعاءها، الدالة ترفض: "للمدير فقط"
```

## ما الذي يبقى كما هو (لا يحتاج تغييراً)

المنطق المحاسبي (القيد المزدوج)، حسابات التأمينات، نظام الترفيع، تصميم الواجهة وألوانها —
كلها سليمة وتُنقل كما هي. التغيير معماري (أين تُخزَّن البيانات وكيف تُحمى)، لا منطقي.

---

## الخلاصة

بعد تنفيذ هذه الخطوات:
- ✅ مصادقة حقيقية لا تُتجاوز
- ✅ كلمات مرور مشفّرة
- ✅ عزل حقيقي بين المدارس (RLS) — حتى لو عُدّل كود المتصفح
- ✅ بيانات دائمة لا تُفقد
- ✅ استعادة كلمة المرور عبر البريد
- ✅ سجل تدقيق غير قابل للتلاعب
- ✅ سير موافقة الرواتب مفروض على الخادم

يبقى الدفع الحقيقي (Thawani/Stripe) كخطوة تالية تحتاج حساب تاجر.
