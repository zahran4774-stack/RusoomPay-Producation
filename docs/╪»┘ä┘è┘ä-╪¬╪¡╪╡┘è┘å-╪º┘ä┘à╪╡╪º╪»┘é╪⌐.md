# دليل تحصين المصادقة — EduPay

معالجة الثغرات الأمنية الحرجة من المراجعة: التحقق من البريد + حدّ المحاولات + CAPTCHA.
معظم هذه الإعدادات في **لوحة Supabase**، والكود مُهيّأ لدعمها.

---

## 1) التحقق من البريد الإلكتروني (حرج)

**المشكلة:** أي شخص يسجّل مدرسة بأي بريد دون إثبات ملكيته → انتحال.

**الحل في لوحة Supabase:**
1. Authentication → Providers → Email
2. فعّل **"Confirm email"** (تأكيد البريد إلزامي)
3. Authentication → Email Templates → عرّب قالب "Confirm signup" إن أردت

**ما فعله الكود:**
- التسجيل الآن يرسل رابط تأكيد ولا يُنشئ المدرسة فوراً
- بيانات المدرسة تُحفظ مؤقتاً في `user_metadata`
- بعد تأكيد البريد وأول دخول، تُنشأ المدرسة تلقائياً
- شاشة "📧 أكّد بريدك" تظهر بعد التسجيل

---

## 2) حدّ المحاولات (Rate Limiting) — حرج

**المشكلة:** لا حدّ لمحاولات تسجيل الدخول → هجمات القوة الغاشمة.

**الحل في لوحة Supabase:**
1. Authentication → Rate Limits
2. اضبط:
   - **Sign in / Sign up**: 5–10 محاولات لكل ساعة لكل IP
   - **Token refresh**: حسب الحاجة
3. Supabase يطبّق حدوداً افتراضية، لكن شدّها للإنتاج

**طبقة إضافية (موصى بها):** فعّل حماية Vercel/Cloudflare ضد الهجمات على مستوى الشبكة.

---

## 3) CAPTCHA على التسجيل والدخول — حرج

**المشكلة:** التسجيل الآلي (bots) ينشئ حسابات وهمية.

**الحل (hCaptcha — مدعوم رسمياً في Supabase):**
1. أنشئ حساباً على hcaptcha.com واحصل على **Site Key** و **Secret Key**
2. لوحة Supabase → Authentication → Settings → Bot and Abuse Protection
3. فعّل **hCaptcha** وألصق الـSecret Key
4. في الكود (مُهيّأ مسبقاً): أزل التعليق عن `captchaToken` في صفحتي الدخول والتسجيل، وأضف ودجت hCaptcha:

```bash
npm install @hcaptcha/react-hcaptcha
```

```tsx
import HCaptcha from '@hcaptcha/react-hcaptcha'
// داخل النموذج:
<HCaptcha sitekey={process.env.NEXT_PUBLIC_HCAPTCHA_SITEKEY!}
  onVerify={(token) => setHcaptchaToken(token)} />
// ثم مرّر captchaToken: hcaptchaToken في options
```

---

## 4) سياسة كلمات المرور (مُطبّقة في الكود)

تم تشديدها في صفحة التسجيل:
- 10 أحرف على الأقل (بدل 8)
- حرف + رقم + رمز إلزامي
- منع كلمات المرور الشائعة (password, 12345678, …)

**طبقة Supabase:** Authentication → Settings → Password → اضبط الحد الأدنى للطول = 10.

---

## 5) وقاية من حصر الحسابات (Account Enumeration)

رسالة الدخول الموحّدة "البريد أو كلمة المرور غير صحيحة، أو لم يُؤكَّد البريد" — لا تكشف إن كان البريد مسجلاً، فيصعب على المهاجم معرفة الحسابات الموجودة. (مُطبّقة في الكود)

---

## 6) المصادقة الثنائية (2FA) — لمدير المنصة

لحساب `platform_admin` (أعلى صلاحية):
1. Authentication → فعّل **MFA (TOTP)**
2. اطلب من مدير المنصة تفعيلها من إعدادات حسابه
3. (متقدّم) افرضها على دور platform_admin عبر سياسة

---

## قائمة التحقق قبل الإطلاق

- [ ] تأكيد البريد مفعّل
- [ ] حدّ المحاولات مضبوط (5–10/ساعة)
- [ ] hCaptcha مفعّل على التسجيل والدخول
- [ ] الحد الأدنى لكلمة المرور = 10
- [ ] 2FA لمدير المنصة
- [ ] قوالب البريد معرّبة

بعد هذه الخطوات، يرتفع تقييم الأمان من 8/10 إلى ~9.5/10، ويصبح النظام جاهزاً أمنياً للإطلاق التجاري.
