# خطة اختبار قبول المستخدم (UAT) — EduPay

اختبار القبول يتحقّق أن النظام يلبّي احتياج المستخدمين الحقيقيين قبل الإطلاق. يُنفّذ يدوياً بواسطة ممثّلين عن كل دور. هذا المستند يُطبع ويُوزّع على المختبرين.

---

## كيفية الاستخدام

1. اختر مختبراً حقيقياً لكل دور (مدير، محاسب، ولي أمر).
2. نفّذ كل سيناريو خطوة بخطوة على بيئة اختبار (لا الإنتاج).
3. سجّل النتيجة: ✅ نجح / ❌ فشل / ⚠️ ملاحظة.
4. أي ❌ يُوثّق بالتفصيل ويُصلح قبل الإطلاق.

**معيار القبول العام:** ينجح الإطلاق إذا اجتاز 100% من السيناريوهات الحرجة (🔴) و≥90% من المهمّة (🟡).

---

## المجموعة 1: مدير المدرسة 🔴

| # | السيناريو | الخطوات | النتيجة المتوقّعة | ✅/❌ |
|---|---|---|---|---|
| 1.1 | تسجيل الدخول | اختر المدرسة → أدخل البريد وكلمة المرور | يدخل للوحة التحكّم بدوره الصحيح | |
| 1.2 | إضافة طالب | لوحة التحكّم → إضافة طالب → املأ البيانات → حفظ | يظهر الطالب في القائمة | |
| 1.3 | إنشاء رسوم | شؤون الطلاب → اختر طالباً → أضف رسماً | يظهر الرسم بالمبلغ والاستحقاق | |
| 1.4 | اعتماد دفعة | الدفعات المعلّقة → اعتمد دفعة ولي أمر | تتحوّل لمدفوعة ويُرحّل القيد | |
| 1.5 | عرض ميزان المراجعة | المحاسبة → ميزان المراجعة | المدين = الدائن (متوازن) | |
| 1.6 | تصدير تقرير | مركز التقارير → اختر تقريراً → تصدير | يُنزّل ملف PDF صحيح | |
