# EduPay — النسخة الإنتاجية (Next.js + Supabase)

تطبيق إدارة مالية للمدارس الخاصة في الخليج، مبني على معمارية آمنة وقابلة للتوسّع.

## ما الذي يحلّه هذا عن النموذج الأولي

| النموذج الأولي (HTML) | النسخة الإنتاجية (هنا) |
|---|---|
| مصادقة في المتصفح (تُتجاوز) | Supabase Auth — JWT على الخادم |
| كلمات مرور كنص صريح | مشفّرة (bcrypt) في Supabase |
| عزل المدارس تجميلي | RLS — عزل مفروض في قاعدة البيانات |
| بيانات في الذاكرة (تُفقد) | PostgreSQL دائمة |

## التشغيل

### 1) المتطلبات
- Node.js 18+
- حساب Supabase (مجاني)

### 2) إعداد قاعدة البيانات
في لوحة Supabase → SQL Editor، نفّذ بالترتيب:
1. `supabase/migrations/01_schema.sql`
2. `supabase/migrations/02_rls_policies.sql`
3. `supabase/migrations/03_functions.sql`

### 3) متغيّرات البيئة
```bash
cp .env.local.example .env.local
# املأ NEXT_PUBLIC_SUPABASE_URL و NEXT_PUBLIC_SUPABASE_ANON_KEY من لوحة Supabase
```

### 4) التثبيت والتشغيل
```bash
npm install
npm run dev
# افتح http://localhost:3000
```

### 5) النشر (Vercel)
```bash
# اربط المستودع بـ Vercel، أضف متغيّرات البيئة، وانشر
```

## البنية

```
app/
  (auth)/login/         ← تسجيل دخول حقيقي
  (auth)/register/      ← تسجيل مدرسة (RPC)
  dashboard/            ← لوحة المعلومات
  students/             ← سجل الطلاب (RLS)
  layout.tsx            ← RTL + الخطوط
lib/
  supabase-client.ts    ← عميل المتصفح
  supabase-server.ts    ← عميل الخادم
middleware.ts           ← حماية المسارات + تحديث الجلسة
supabase/migrations/    ← المخطط + RLS + الدوال
```

## الأمان — كيف يعمل

1. **المصادقة**: Supabase Auth يتحقق على الخادم ويصدر JWT في كوكي آمن
2. **الحماية**: `middleware.ts` يمنع الوصول للمسارات المحمية دون تسجيل دخول
3. **العزل**: كل استعلام يمرّ عبر RLS — يستحيل رؤية بيانات مدرسة أخرى
4. **الأدوار**: السياسات تفرض صلاحيات الدور (مثلاً: اعتماد الرواتب للمدير فقط)

## الخطوات التالية

- نقل بقية الشاشات (الموظفون، الرواتب، الفواتير، المحاسبة) من النموذج
- ربط بوابة دفع حقيقية (Thawani لعُمان / Stripe)
- رفع الشعارات والإيصالات إلى Supabase Storage
- إضافة طبقة مدير المنصة (super-admin) لاعتماد التحويلات البنكية

المنطق المحاسبي وحسابات التأمينات والتصميم تُنقل كما هي من النموذج الأولي —
التغيير معماري (الأمان والثبات)، لا منطقي.
